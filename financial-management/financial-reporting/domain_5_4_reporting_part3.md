### Data Integration Architecture

**Overall Flow:**
```
Marketplaces        Inventory         Accounting
└─ APIs ────┐       └─ TradeGecko ────┐
            └──> BigQuery Data Warehouse ──> Looker (BI)
                      ↑                           ↓
                      └──── Fatture in Cloud ────┘
                            (GL master)
```

**Implementation:**
1. **Marketplace APIs → BigQuery** (nightly, 10 PM)
   - Amazon MWS, eBay API, Cdiscount API
   - Data: Orders, returns, fees, reconciliation reports
   - Transformation: Denormalization, currency conversion, deduplication

2. **Fatture in Cloud → BigQuery** (nightly, 11 PM)
   - GL export (trial balance, subledger)
   - Manual upload via CSV (month-end accruals, adjusting entries)
   - Historical: 3-year rolling GL

3. **BigQuery → Looker** (hourly dashboard refresh)
   - Core tables: Revenue (daily), COGS, OpEx, Inventory
   - Derived metrics: Margin %, KPIs, variance

4. **Auto-Posted Accruals** (via Fatture in Cloud API)
   - Chargeback/return accrual (weekly, algorithm-based)
   - Marketplace fee true-up (weekly)
   - Manual review: Monthly by Controller before closing

5. **Inventory Valuation** (weekly)
   - FIFO method: Cost flow from purchase orders
   - Obsolescence reserve: Aging analysis (>6 months flagged)
   - Write-down GL entries: Manual via checklist

- [ ] Asset register (description, acquisition date, cost, accumulated depreciation)

- [ ] Accrued expenses: Payroll (€2-3K/month), marketplace chargebacks/returns (€10-15K/month)
